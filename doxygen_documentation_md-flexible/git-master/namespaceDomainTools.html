Generates a decomposition with a specific number of subdomains.

This function uses prime factorization to determine the number of subdomains in each dimension. Subdivision of a dimension can be prevented using the subdivideDimension argument. By preventing subdivision in a dimension one can achieve different grid layouts (e. g. collumn layout)

Parameters
subdomainCountThe number of subdomains in the resulting decomposition.
subdivideDimensionDecides if a dimension will be subdivided.
decompositionArray containing the number of subdomains per dimension.

Returns the accumulated tail of a decomposition.

The accumulated tail refers to the number of elements in a decomposition's tail. If the decomposition is [4,3,2], the accumulated tail of the subdivision starting after index 0 is 3 * 2 = 6. The accumulated tail of this decomposition starting after index 1 is 2 . the accumulated tail of this decomposition starting after index 2 is 1.

Returns the extent of a subdomain in a domain decomposition.

The extent is defined by the starting coordinates of the subdomain within the domain decomposition. A subdomain with id [2, 3, 4] in a subdivision [5, 5, 5] has the corresponding extent [2, 3, 3, 4, 4, 5], where the values are defined as follows: [ xmin, xmax, ymin, ymax, zmin, zmax ]. The global domain extent of this domain decomposition would be [ 0, 5, 0, 5, 0, 5 ].

Parameters
subdomainIndexThe index of the subdomain for which to calculate the domain extent.
decompositionThe decomposition subdomain's respective global domain.
Returns
the extent of the subdomain with index subdomainIndex.
