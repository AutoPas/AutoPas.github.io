Detailed Description

Interface to the Active Harmony (AH) tuning framework.

If a global AH server is provided and MPI is enabled, the server can be reached via the environment variables HARMONY_HOST and HARMONY_PORT. They have to be set to the host address and port of the server respectively.

bool autopas::ActiveHarmony::tune ( bool  currentInvalid)
inlineoverridevirtual

Selects the next configuration to test or the optimum.

A bool is returned indicating whether more tuning steps are required (=true) or the optimum was found (=false). The new configuration can be obtained by getCurrentConfiguration. It is the configuration which is either the next configuration to test (=true) or the optimum (=false).

Parameters
currentInvalidTells the tune() function that the currently selected configuration is invalid. This can be used to avoid getting stuck in an invalid optimum.
Returns
false iff new configuration is the selected optimum.
